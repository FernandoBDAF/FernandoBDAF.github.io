Create Your Page

  1. Go to GitHub Page, you can access it through the following link: https://github.com/.
  2. Create your account (Sign up) or access it (Sign in) if you already created it before.
  3. Now that you are logged in, on the up-left corner of the page click the NEW button, so you can create the repository of your page.
  4. Fill the *Repository name* with the following pattern: your account name + .github.io - when you do that, this repository will automatically host your personal page. For example, my account name is FernandoBDAF and my personal page is FernandoBDAF.github.io.
  5. Navigate to your repository and hit the "Add file" button (in the high-middle of the page) and choose Upload files. You can drag the index.html file we are going to create in the next step.
  6. Open the folder of the project in VS Code and select file -> new file. Name it "index.html" and write the code you see in the image behind making the appropriate changes. Drag the photo you want to have on your page to the same folder and update the src tag.
  7. Now you are good to go, access your page! The address will be the exact name of the repository, look mine as an example: FernandoBDAF.github.io

The use of GIT in the development process

What branchs we are going to use

We are going to work in the GIT FLOW as standart. We want to have the following branchs

  • Main: production
  • Develop: we put the features here to test before comminting to main
  • Feature: for each new feature we create a branch with the name feature/NAME. After the work is done here, we commit to develop to test before commiting to main
  • Between Main and Release we may have:
    • Hotfix: when we have bugs we create branchs with name hotfix/NAME. After the work is done we commit it to both main and develope (because both have to be working well)
    • Release: this is where we test before sending to develop

Creatin Branchs

  1. git branch: show all branchs
  2. git branch feature/NAME:
    3. create the branch
  3. git switch [branch_name]: change branch
  4. :

You work with the branch, adding and commiting until you done with your new feature

Commiting a new feature

  1. Merge the feature to the branch develop
    1. git checkout develop: Change Branch
    2. git merge feature/NAME: bring the changes to the develop branch
    3. If you have any conflits, you have to fix it and then git add . and git merge --continue.
    4. git add .: Add files
    5. git push origin master: Update the online version
    6. git commit -m "Merge feature into develop": Commit
  2. Remove the branch with this feature
    3. git branch -d feature/NAME: Delete the branch, if there are not commited work you need to use -D. To remove a remote branch, use git push origin --delete Remote_branch_name
